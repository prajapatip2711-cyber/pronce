# PRONCE

A Vercel-ready Next.js 15 storefront for premium products and digital services. It uses Google Sheets through Google Apps Script for orders, enquiries and customers—no database service is required.

## Run locally

```bash
npm install
copy .env.example .env.local
npm run dev
```

Visit `http://localhost:3000`. Add the environment variables described below before submitting an order or enquiry.

## Google Sheets + Apps Script

1. Create a Google Sheet for PRONCE.
2. Open **Extensions → Apps Script**, replace the default file with [`google-apps-script/Code.gs`](google-apps-script/Code.gs), save it, and run `setup` once. Approve the Google authorization prompt; it creates all required tabs.
3. Click **Deploy → New deployment → Web app**. Select *Execute as: Me* and *Who has access: Anyone* (or restrict it to the account that hosts the site when using a proxy).
4. Copy the Web App URL into `GOOGLE_APPS_SCRIPT_URL` in Vercel and `.env.local`.

The storefront sends JSON payloads to `Physical Orders`, `Customers` and `Enquiries`. Apps Script timestamps each record; the Order ID is generated by the Next.js API.

## Environment variables

| Name | Purpose |
| --- | --- |
| `GOOGLE_APPS_SCRIPT_URL` | Apps Script Web App deployment URL |
| `ADMIN_PASSWORD` | Strong password for the `/admin` sign-in gate |
| `NEXT_PUBLIC_SITE_URL` | Canonical production URL, e.g. `https://pronce.vercel.app` |

Never commit `.env.local`. Use Vercel’s **Settings → Environment Variables** for production values.

## Deploy to Vercel

1. Push this directory to a new GitHub repository.
2. Import that repository at [Vercel](https://vercel.com/new).
3. Add all three environment variables.
4. Deploy. Vercel detects Next.js automatically; no build override is needed.

```bash
git init
git add .
git commit -m "Build PRONCE storefront"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/pronce.git
git push -u origin main
```

## Operations

- Products and portfolio items are currently curated in `lib/data.ts`, keeping the public catalog fast and type-safe.
- Google Sheets is the operational source for orders, customers, inventory, enquiries and logs. Filter/order updates are managed there and reflected in fulfilment communications.
- Product images use remote optimized images; the supplied Instagram QR is stored at `public/images/instagram-qr.png`.
